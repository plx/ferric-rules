# ferric-rules Landing Page Brief

## Page Goal

Create a serious, documentation-forward landing page for `ferric-rules`: a mostly CLIPS-compatible forward-chaining rules engine in Rust, designed for easy embedding in modern applications.

## Primary Message

CLIPS-style rules, modern Rust embedding.

## Supporting Messages

- Mostly CLIPS-compatible semantics for practical rule sets.
- Independent engine instances instead of global runtime state.
- Normal Rust crate integration with FFI and binding paths.
- Built around Rete, templates, modules, salience, and common CLIPS standard-library behavior.
- Early but functional, with clear compatibility documentation.

## Expected Navigation

- Guide
- Compatibility
- Migration
- Performance
- Bindings
- GitHub

## Content Blocks

- Hero with concise product position and code-forward CTA.
- Embedded example showing `Engine::with_rules`, assertions, and `run`.
- Three value pillars: Compatibility, Embeddability, Rigor.
- Documentation card grid with links to existing docs.
- Honest implementation status and known exclusions.
- Footer with license, repository, and documentation links.

## Visual Requirements

- Serious and minimal enough for repeated documentation use.
- Interesting first impression through architectural drafting and steel-girder imagery.
- First viewport must show the product name prominently and leave a hint of the next section visible.
- Avoid generic SaaS hero cards, one-note orange palettes, and visual clutter.
